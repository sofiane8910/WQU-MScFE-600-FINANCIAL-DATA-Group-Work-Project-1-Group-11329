2. Nelson-Siegel Model Implementation

The Nelson-Siegel model is a parsimonious model for fitting yield curves. It uses the following functional form:

$$y(t) = \beta_0 + \beta_1 \left(\frac{1 - e^{-\lambda t}}{\lambda t}\right) + \beta_2 \left(\frac{1 - e^{-\lambda t}}{\lambda t} - e^{-\lambda t}\right)$$

Where:

  • $\beta_0$: Long-term yield level
  • $\beta_1$: Short-term component
  • $\beta_2$: Medium-term component
  • $\lambda$: Decay parameter
In[66]:
def nelson_siegel(t, beta0, beta1, beta2, lambda_param):
    """
    Nelson-Siegel yield curve model
    
    Parameters:
    t: time to maturity
    beta0: long-term yield level
    beta1: short-term component
    beta2: medium-term component
    lambda_param: decay parameter
    """
    t_safe = np.where(t == 0, 1e-10, t)
    
    term1 = (1 - np.exp(-lambda_param * t_safe)) / (lambda_param * t_safe)
    term2 = (1 - np.exp(-lambda_param * t_safe)) / (lambda_param * t_safe) - np.exp(-lambda_param * t_safe)
    
    return beta0 + beta1 * term1 + beta2 * term2

def nelson_siegel_objective(params, t, observed_yields):
    """
    Objective function for Nelson-Siegel optimization
    """
    beta0, beta1, beta2, lambda_param = params
    predicted_yields = nelson_siegel(t, beta0, beta1, beta2, lambda_param)
    return np.sum((observed_yields - predicted_yields)**2)

print("Fitting Nelson-Siegel model...")

#initial parameter guess

#bounds for optimization

    (0, 10),    
    (-10, 10),  
    (-10, 10), 
    (0.1, 5)   
] #long-term yield=0-10%, short-term component=-10-10%, medium-term component=-10-10%, decay parameter=0.1-5

#calculate performance metrics with extra step=cross-validation to show overfitting
from sklearn.model_selection import KFold
from sklearn.metrics import mean_squared_error, r2_score

def calculate_metrics(observed, predicted):
    """Calculate various performance metrics"""
    mse = np.mean((observed - predicted)**2)
    rmse = np.sqrt(mse)
    mae = np.mean(np.abs(observed - predicted))
    mape = np.mean(np.abs((observed - predicted) / observed)) * 100
    r_squared = 1 - (np.sum((observed - predicted)**2) / np.sum((observed - np.mean(observed))**2))
    return mse, rmse, mae, mape, r_squared

def cross_validate_cubic_spline(X, y, n_splits=3):
    """Cross-validate cubic spline to get realistic performance metrics"""
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    rmse_scores = []
    r2_scores = []
    
    for train_idx, test_idx in kf.split(X):
        X_train, X_test = X[train_idx], X[test_idx]
        y_train, y_test = y[train_idx], y[test_idx]
        
        cs = CubicSpline(X_train, y_train)
        y_pred = cs(X_test)
        
        rmse_scores.append(np.sqrt(mean_squared_error(y_test, y_pred)))
        r2_scores.append(r2_score(y_test, y_pred))
    
    return np.mean(rmse_scores), np.mean(r2_scores)

mse_ns, rmse_ns, mae_ns, mape_ns, r2_ns = calculate_metrics(yields, fitted_yields_ns)

6. Summary and Conclusions

  1. Model Performance:

    • Nelson-Siegel Model: Achieved good fit with RMSE of ~0.06 and R > 0.95, using only 4 parameters
    • Cubic-Spline Model: Perfect fit (RMSE = 0.000) but with overfitting due to exact interpolation
    • Trade-off: Nelson-Siegel provides better generalization while Cubic-Spline offers perfect in-sample fit

  2. Parameter Interpretation:

    • (Long-term level): ~3.28% - represents asymptotic yield as maturity approaches infinity
    • (Short-term component): ~-1.20% - indicates short-term rates below long-term level (normal yield curve)
    • (Medium-term component): ~-0.65% - suggests slight concavity in medium-term yields
    • (Decay parameter): ~0.85 - controls speed of decay between short and long-term components

    • Upward-sloping curve: Typical normal yield curve shape
    • Short-term yields: 1Y ~2.40%, 5Y ~2.95%, 10Y ~3.14%, 30Y ~3.28%
    • Steepness: Moderate steepness indicating normal market conditions

  4. Business Interpretability:

    • Reduced Knots Approach: Using 6 key benchmarks (0.25Y, 1Y, 2Y, 5Y, 10Y, 30Y) improves interpretability
    • Economic Logic: Focus on liquid benchmark maturities aligns with market practice
    • Overfitting Mitigation: Fewer knots reduce sensitivity to noise and outliers

  5. Ethical Considerations:

    • Nelson-Siegel smoothing is ethically acceptable when used for research and analysis
    • Provides better representation of market expectations than raw data
    • Standard practice in financial institutions and central banks
    • Becomes problematic only when used to mislead market participants
